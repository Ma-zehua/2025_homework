Git常用命令

一、仓库初始化与克隆
- git init：初始化本地仓库（生成.git文件夹）
- git clone <远程仓库地址>：克隆远程仓库到本地

二、文件暂存与提交
- git add <文件名>：添加单个文件到暂存区
- git add <文件夹>：添加整个文件夹到暂存区
- git add .：添加当前目录所有未忽略文件到暂存区
- git add -u：仅添加已跟踪文件的修改到暂存区
- git commit -m "提交说明"：提交暂存区内容到本地仓库
- git commit -am "说明"：直接提交已跟踪文件的修改（跳过暂存区）
- git commit --amend：修改最近一次提交（补充或修正信息）

三、查看状态与差异
- git status：查看工作区与暂存区状态（修改、新增、删除文件）
- git diff：查看工作区与暂存区的差异
- git diff <文件名>：查看单个文件的差异
- git diff --staged：查看暂存区与本地仓库的差异
- git diff <分支1> <分支2>：对比两个分支的差异

四、分支操作
- git branch：查看本地分支（当前分支前带*）
- git branch -r：查看远程分支
- git branch -a：查看所有分支（本地+远程）
- git branch <分支名>：创建分支
- git checkout <分支名>：切换到指定分支
- git checkout -b <分支名>：创建并切换到新分支
- git switch <分支名>：切换分支（新版推荐）
- git switch -c <分支名>：创建并切换新分支（新版推荐）
- git merge <目标分支>：将目标分支合并到当前分支
- git branch -d <分支名>：删除已合并的本地分支
- git branch -D <分支名>：强制删除未合并的本地分支
- git push origin --delete <分支名>：删除远程分支

五、远程仓库交互
- git remote：列出远程仓库别名（默认通常为origin）
- git remote -v：查看远程仓库实际地址
- git remote add origin <远程仓库地址>：关联本地仓库与远程仓库
- git pull：拉取远程分支最新内容并合并到本地
- git pull origin <远程分支>：拉取指定远程分支并合并
- git push origin <本地分支>：推送本地分支到远程仓库
- git push -u origin <分支>：首次推送时关联远程分支（后续可直接git push）

六、撤销与回退
- git restore <文件名>：放弃工作区修改，恢复到最近提交/暂存状态
- git restore .：放弃当前目录所有未暂存的修改
- git restore --staged <文件名>：将已add的文件从暂存区撤回（保留工作区修改）
- git reset --hard <提交ID>：彻底回退到指定版本（工作区、暂存区同步）
- git reset --soft <提交ID>：回退但保留工作区和暂存区修改（用于重新提交）

七、查看提交历史
- git log：显示完整提交历史（按时间倒序）
- git log --oneline：简化显示
- git log -n <数字>：显示最近n次提交

八、暂存工作区（临时切换场景）
- git stash：暂存工作区所有修改
- git stash pop：恢复最近一次暂存内容，并删除暂存记录
- git stash list：查看所有暂存记录
- git stash drop：删除最近一次暂存记录
